The `Loader` component provides visual feedback to users during processing or loading states, helping maintain engagement and communicate that the system is working. Typical use cases include:

- Indicating content or data is being loaded (e.g., product listings, search results).
- Showing processing status during form submissions or checkout.
- Displaying loading state within buttons during async actions (e.g., `Add to Cart`).

## Usage

### Import the component

```tsx
import { Loader } from "@faststore/ui";
```

### Import styles

To apply styles, include the following in your stylesheet:

```scss
@import "@faststore/ui/src/components/atoms/Loader/styles.scss";
```

For details, see [Importing FastStore UI component styles](https://developers.vtex.com/docs/guides/faststore/using-themes-importing-ui-components-styles).

### Design tokens: Dark

The dark variant (default) uses a darker color scheme, making it suitable for use on light backgrounds and most standard interface contexts.

The light variant uses a lighter color scheme, ideal for displaying loaders on dark backgrounds or in high-contrast scenarios.

## Data attributes

You can target and override `Loader` styles using the following data attributes:

`data-fs-loader`

`data-fs-loader-variant="light" | "dark"`
