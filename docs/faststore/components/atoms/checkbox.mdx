Checkboxes are interactive form controls that let users select one or more options by toggling them on or off. They are commonly used in forms, filter panels, and preference settings.

The `Checkbox` component provides accessible markup, design tokens for customization, and states for `checked`, `disabled`, and partially (`partial`) checked usage.

## Usage

### Import the component

```tsx
import { Checkbox } from '@faststore/ui'
```

### Import styles

To apply styles, include the following in your stylesheet:

```scss
@import '@faststore/ui/src/components/atoms/Checkbox/styles.scss';
```

For details, see [Importing FastStore UI component styles](https://developers.vtex.com/docs/guides/faststore/using-themes-importing-ui-components-styles).

## Best practices

### ✅ Do's

- Always pair a Checkbox with a descriptive [Label](https://developers.vtex.com/docs/guides/faststore/atoms-label) to improve accessibility and clarify the option’s purpose.

### ❌ Don’ts

- Don’t use a `Checkbox` when only one option is possible. In this case, use a [Radio](https://developers.vtex.com/docs/guides/faststore/atoms-radio) instead.
- Don’t use checkboxes for actions that happen immediately on click (e.g., submitting data)—they should only toggle state.
