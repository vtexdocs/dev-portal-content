The `SKUMatrix` compound surfaces complex product variations in a single, high-density view. Typical use cases include:

- Ordering multiple SKU combinations such as workwear jackets across sizes and colors without hopping between PDPs.
- Offering quick actions relevant for B2B procurement (bulk discounts, pack sizes) next to the Add to Cart summary.

## Import

Import the compound from [@faststore/ui](https://developers.vtex.com/docs/guides/faststore/components-index).

```tsx
import {
  SKUMatrix,
  SKUMatrixTrigger,
  SKUMatrixSidebar,
  useSKUMatrix,
} from "@faststore/ui";
```

### Import styles

Add the sidebar styling tokens and layout rules to your theme stylesheet:

```scss
@import "@faststore/ui/src/components/organisms/SKUMatrix/styles.scss";
```

Follow the [Importing FastStore UI component styles](https://developers.vtex.com/docs/guides/faststore/using-themes-importing-ui-components-styles) instructions. The quantity selector, badges, and table also rely on their respective partials.

## Usage

The matrix relies on context state managed by `SKUMatrix`. Make sure your app is wrapped with `UIProvider` (for toasts) and provide the variant data before rendering the sidebar.

```tsx
import React, { useEffect } from "react";
import {
  SKUMatrix,
  SKUMatrixTrigger,
  SKUMatrixSidebar,
  useSKUMatrix,
} from "@faststore/ui";

const warehouseVariants = [
  {
    id: "sku-45",
    name: "Navy · Size 40",
    image: {
      url: "/img/navy-40.png",
      alternateName: "Safety boot navy size 40",
    },
    inventory: 24,
    availability: "inStock",
    price: 75,
    listPrice: 90,
    priceWithTaxes: 75,
    listPriceWithTaxes: 90,
    specifications: { color: "Navy", size: "40", safetyrating: "S3" },
    selectedCount: 0,
    offers: {
      /* ...commerce data omitted for brevity... */
    },
  },
  // other variant products...
];

function SKUMatrixContent() {
  const { setAllVariantProducts } = useSKUMatrix();

  useEffect(() => {
    setAllVariantProducts(warehouseVariants);
  }, []);

  return (
    <>
      <SKUMatrixTrigger variant="secondary">Plan restock</SKUMatrixTrigger>

      <SKUMatrixSidebar
        title="Select safety boot quantities"
        direction="rightSide"
        size="partial"
        loading={false}
        columns={{
          name: "Variation",
          additionalColumns: [
            { label: "Color", value: "color" },
            { label: "Size (EU)", value: "size" },
            { label: "Safety rating", value: "safetyrating" },
          ],
          availability: {
            label: "Availability",
            stockDisplaySettings: "showStockQuantity",
          },
          price: "Price",
          quantitySelector: "Quantity",
        }}
        buyProps={{
          "data-testid": "bulk-add",
          "data-sku": "safety-boot",
          "data-seller": "seller-1",
          onClick: () => {
            /* call cart mutation */
          },
        }}
        overlayProps={{ "aria-label": "SKU matrix overlay" }}
      />
    </>
  );
}

export default function ProductPageSkuMatrix() {
  return (
    <SKUMatrix data-testid="sku-matrix">
      <SKUMatrixContent />
    </SKUMatrix>
  );
}
```

## Best practices

### ✅ Do's

- Use concise variation labels (e.g., "Black - 42") so the table stays scannable on partial-width slide-overs.
- Keep price and availability in sync with your store OMS; show a toast when quantity adjustments are coerced.
- Localize both column headers and toast messages for multinational catalogs.

### ❌ Don'ts

- Don't expose the matrix when a product is single-SKU; it adds unnecessary friction.
- Don't rely on color alone for stock communication pair badge colors with descriptive text.
- Don't allow quantities beyond available stock; the component exposes validation hooks for this reason.
- Don't hardcode dimensions; merchants should be able to adjust columns per category via Headless CMS.
