---
title: 'Enums'
---

Enums define a fixed set of allowed values for a field or argument in the FastStore GraphQL schema. When a field type or query argument is an enum, only the values listed for that enum are accepted. They are used as argument values in queries (e.g., `sort`) and as fields returned in query responses (e.g., `type` in a collection).

## StoreCollectionType

Product collection type, returned in the `type` field of the [`StoreCollection`](https://developers.vtex.com/docs/guides/faststore/schema-objects#storecollection) object. These values reflect the VTEX Catalog hierarchy: departments, categories, and subcategories make up the catalog tree, while brands, clusters, and collections are additional grouping mechanisms configured in the VTEX Admin.

> ℹ️ To learn more about how the VTEX catalog is structured, see [Categories](https://help.vtex.com/docs/tutorials/what-is-a-category), [Brands](https://help.vtex.com/en/tutorial/what-is-a-brand), and [Product collection](https://help.vtex.com/en/docs/tutorials/creating-a-product-collection) guides.

<table>
<thead>
<th align="left">Value</th>
<th align="left">Description</th>
</thead>
<tbody>
<tr>
<td valign="top"><strong>Department</strong></td>
<td>

Root level of the VTEX catalog category tree. Departments are the broadest product grouping (e.g., `Electronics`, `Clothing`). In the VTEX Admin catalog, this corresponds to a first-level category.

</td>
</tr>
<tr>
<td valign="top"><strong>Category</strong></td>
<td>

Second level of the VTEX catalog category tree, nested inside a Department (e.g., `Smartphones` inside `Electronics`). In the VTEX Admin catalog, this corresponds to a second-level category.

</td>
</tr>
<tr>
<td valign="top"><strong>SubCategory</strong></td>
<td>

Third level of the VTEX catalog category tree, nested inside a Category (e.g., `Android Phones` inside `Smartphones`). In the VTEX Admin catalog, this corresponds to a third-level category.

</td>
</tr>
<tr>
<td valign="top"><strong>Brand</strong></td>
<td>

A product brand registered in the VTEX catalog. Brands are configured in the VTEX Admin under **Catalog > Brands**.

</td>
</tr>
<tr>
<td valign="top"><strong>Cluster</strong></td>
<td>

A custom product grouping configured in the VTEX Admin (e.g., `Best Sellers`, `New Arrivals`). Unlike categories, clusters do not follow a fixed hierarchy and can group products from different departments or brands. Clusters are configured in the VTEX Admin under **Catalog > Product Clusters**.

</td>
</tr>
<tr>
<td valign="top"><strong>Collection</strong></td>
<td>

A manually curated product group created in the VTEX Admin (e.g., `Summer Sale`). Collections are used to organize products for promotional or merchandising purposes and are configured under **Catalog > Collections**.

</td>
</tr>
</tbody>
</table>

### Usage example

The following query retrieves a collection by its slug and returns its `type`, which resolves to one of the `StoreCollectionType` values above:

```graphql
query {
  collection(slug: "electronics") {
    id
    slug
    type
    seo {
      title
      description
    }
  }
}
```

## StoreFacetType

Search facet type, returned in search results to indicate what kind of input control a facet uses. Used in [`StoreFacet`](https://developers.vtex.com/docs/guides/faststore/schema-objects#storefacet) objects to describe the filter options available on a search results page.

<table>
<thead>
<th align="left">Value</th>
<th align="left">Description</th>
</thead>
<tbody>
<tr>
<td valign="top"><strong>BOOLEAN</strong></td>
<td>

Indicates a filter with boolean options (e.g., `yes` and `no`), for narrowing search results.

</td>
</tr>
<tr>
<td valign="top"><strong>RANGE</strong></td>
<td>

Indicates a filter to choose between a range of values (e.g., `price`, `size`, etc) to refine search results.

</td>
</tr>
</tbody>
</table>

## StoreSort

Product search results sorting options. Passed as the `sort` argument in the [`search` query](https://developers.vtex.com/docs/guides/faststore/schema-queries#search) to control how results are ordered.

### Usage example

The following example passes `price_asc` as the `sort` argument to return search results sorted by lowest price first:

```graphql
query {
  search(first: 10, sort: price_asc, term: "sneakers") {
    products {
      edges {
        node {
          name
          offers {
            lowPrice
          }
        }
      }
    }
  }
}
```

## StoreStatus

Status used to indicate a message type. Returned in cart and notification messages (e.g., [`StoreNotificationItem`](https://developers.vtex.com/docs/guides/faststore/schema-objects#storenotificationitem)) to indicate the severity of a user-facing message.

<table>
<thead>
<th align="left">Value</th>
<th align="left">Description</th>
</thead>
<tbody>
<tr>
<td valign="top"><strong>INFO</strong></td>
<td>Indicates non-critical notifications or informative messages related to the user's action.</td>
</tr>
<tr>
<td valign="top"><strong>WARNING</strong></td>
<td>Indicates a potential issue or unexpected behavior that might require user attention but doesn't necessarily prevent them from continuing.</td>
</tr>
<tr>
<td valign="top"><strong>ERROR</strong></td>
<td>Indicates a critical issue that prevents the user from completing an action or indicates a problem with the system.</td>
</tr>
</tbody>
</table>
