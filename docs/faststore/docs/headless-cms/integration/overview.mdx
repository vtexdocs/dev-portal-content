> ⚠️ New FastStore projects use the [CMS](https://developers.vtex.com/docs/guides/cms-for-faststore-storefronts) configured during [WebOps onboarding](https://developers.vtex.com/docs/guides/faststore/getting-started-2-starting-the-project#cms-integration). These guides apply only to stores still on Headless CMS (legacy).

This section walks you through the full Headless CMS (legacy) integration workflow for FastStore: connect your storefront to the CMS, define Content Types and Sections, and promote your changes to production so editors can manage content in the VTEX Admin.

Follow the guides below in order.
