## Adding a custom font

Using a custom font helps to keep a cohesive brand across your store. To maintain readability in your store, choose a legible font, appropriate colors, and a suitable text size.
 
In FastStore, there are two ways to load a custom font, depending on your font choice:
 
- [Self-hosted Inter font (experimental)](#self-hosted-inter-font-experimental): Recommended for stores that use Inter and want to improve performance.
- [External font via WebFonts.tsx](#loading-an-external-font-via-webfontstsx): Standard approach for any other font family.

### Self-hosted Inter font (experimental)
 
> ⚠️ This feature is experimental and is available from FastStore [v4.2.0](/TBD) onwards and only applies to stores using the **Inter** font family.
 
If your store uses Inter, you can self-host it directly via FastStore instead of loading it from Google Fonts. This eliminates the render-blocking request to `fonts.googleapis.com`, which can improve First Contentful Paint (FCP) by approximately 660ms on mobile.
 
To enable it, follow the steps below:
 
1. In your `discovery.config.js`, add the `experimental.optimizedFonts` flag:

   ```js
   experimental: {
     optimizedFonts: true,
   }
   ```
 
2. Remove the Inter `<link>` tag from `customizations/src/fonts/WebFonts.tsx` to avoid loading the font twice:

   ```tsx
   // Remove or comment out the line below
   // <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700;900&display=swap" />
   ```
 
3. In your custom theme file, usually `src/themes/custom-theme.scss`, make sure the `--fs-text-face-body` token is set to Inter (if it isn't already):

   ```scss
   @layer theme {
     .theme {
       --fs-text-face-body: 'Inter', -apple-system, system-ui, BlinkMacSystemFont, sans-serif;
     }
   }
   ```
 
4. Run `yarn dev` and check the browser to confirm Inter is loading from your own domain, not from `fonts.googleapis.com`.

> ℹ️ If your store uses a font other than Inter, use the [Loading an external font via WebFonts.tsx](#loading-an-external-font-via-webfontstsx) approach.
 
### Loading an external font via WebFonts.tsx
 
This is the standard approach for loading any font family via an external provider such as Google Fonts. Follow the steps below to add a custom font to your store.
 
1. Go to the `src` folder and create the `fonts` folder.
2. Inside the `src/fonts` folder, create a new `WebFonts.tsx` file and add a `<link>` tag for your `font-family` of choice as in the example below.

   ```tsx
   /* eslint-disable @next/next/no-page-custom-font */
 
   function WebFonts() {
     return (
       <>
         {/* Add a <link> tag for your font-family of choice */}
         <link
           rel="stylesheet"
           href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700;900&display=swap"
         />
       </>
     );
   }
 
   export default WebFonts;
   ```
 
3. In your custom theme file, usually `src/themes/custom-theme.scss`, use the `--fs-text-face-body` token with the `font-family` you chose.

   > ℹ️ We recommend using [fallback fonts](https://developer.chrome.com/blog/font-fallbacks/#background) as system fonts to ensure quick, consistent, and readable content.

   ```scss
   @layer theme {
     .theme {
       // --------------------------------------------------------
       // Typography (Branding Core)
       // --------------------------------------------------------
       --fs-text-face-body: 'Inter', -apple-system, system-ui, BlinkMacSystemFont, sans-serif;
     }
   }
   ```
 
4. Run `yarn dev` and check the browser to see the new `font-family` applied.
